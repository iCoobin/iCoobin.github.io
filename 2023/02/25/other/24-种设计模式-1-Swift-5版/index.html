24 种设计模式-1 (Swift 5版)

创建型模式（Creational Patterns）

1. 简单工厂模式（Simple Factory Pattern）

简单工厂模式（Simple Factory Pattern）是一种创建型设计模式，它的主要目的是提供一个统一的接口来创建对象，而无需暴露具体的创建逻辑。该模式常用来解决创建对象时客户端代码需要知道所有可能被创建对象类的问题。

在简单工厂模式中，通常包括一个工厂类和多个产品类。工厂类根据客户端请求的参数来决定创建哪个产品类的对象，并返回给客户端。这样客户端就无需直接调用产品类的构造函数来创建对象了。

以下是一个使用简单工厂模式的示例代码：

  • Swift Code

    // 定义一个接口，所有产品类都要实现这个接口
protocol Product {
    func use()
}

// 具体的产品类实现 Product 接口
class ProductA: Product {
    func use() {
        print("ProductA used")
    }
}

class ProductB: Product {
    func use() {
        print("ProductB used")
    }
}

// 简单工厂类
class SimpleFactory {
    enum ProductType {
        case A, B
    }

    static func createProduct(type: ProductType) -> Product {
        switch type {
        case .A:
            return ProductA()
        case .B:
            return ProductB()
        }
    }
}

// 客户端代码
let productA = SimpleFactory.createProduct(type: .A)
productA.use() // 输出: "ProductA used"

let productB = SimpleFactory.createProduct(type: .B)
productB.use() // 输出: "ProductB used"

在上面的示例代码中，**Product** 定义了一个使用产品的接口。**ProductA** 和 ProductB 是具体的产品类，它们都实现了 Product 接口。**SimpleFactory** 是简单工厂类，它根据客户端请求的参数来创建不同的产品类的对象，然后返回给客户端。最后，客户端通过调用返回的对象的 use 方法来使用产品。

2. 抽象工厂模式（Abstract Factory Pattern）

抽象工厂模式（Abstract Factory Pattern）是一种创建型设计模式，它提供了一种创建一系列相关或相互依赖对象的接口，而无需指定它们的具体类。这种模式属于对象创建型模式，它解决了在不指定对象具体类的情况下创建一系列相关或相互依赖的对象问题。

抽象工厂模式通常用于以下场景：

  • 一个系统要独立于它的产品的创建、组合和表示时
  • 一个系统要由多个产品系列中的一个来配置时
  • 强调一系列相关的产品对象的设计以便进行联合使用时
  • 提供一个产品类库，只想显示它的接口而不是实现时

下面是一个简单的 Swift 5 示例，展示了如何使用抽象工厂模式创建和使用不同系列的产品对象。假设我们需要创建一组不同的计算机，每台计算机包含 CPU 和主板，分为 Intel 和 AMD 两个系列。我们可以定义一个 AbstractFactory 协议，其中包含了创建 CPU 和主板的两个方法，然后定义两个具体的工厂类 IntelFactory 和 AMDFactory 分别实现该协议，以便创建相应系列的产品。最后，我们可以在客户端中使用工厂对象来创建所需的产品。

  • Swift Code

    // 抽象产品协议
protocol CPU {
    var name: String { get }
}

protocol Motherboard {
    var name: String { get }
}

// 抽象工厂协议
protocol AbstractFactory {
    func createCPU() -> CPU
    func createMotherboard() -> Motherboard
}

// 具体产品
class IntelCPU: CPU {
    var name: String {
        return "Intel CPU"
    }
}

class IntelMotherboard: Motherboard {
    var name: String {
        return "Intel Motherboard"
    }
}

class AMDCPU: CPU {
    var name: String {
        return "AMD CPU"
    }
}

class AMDMotherboard: Motherboard {
    var name: String {
        return "AMD Motherboard"
    }
}

// 具体工厂
class IntelFactory: AbstractFactory {
    func createCPU() -> CPU {
        return IntelCPU()
    }

    func createMotherboard() -> Motherboard {
        return IntelMotherboard()
    }
}

class AMDFactory: AbstractFactory {
    func createCPU() -> CPU {
        return AMDCPU()
    }

    func createMotherboard() -> Motherboard {
        return AMDMotherboard()
    }
}

// 客户端代码
let intelFactory = IntelFactory()
let amdFactory = AMDFactory()

let intelCPU = intelFactory.createCPU()
let intelMotherboard = intelFactory.createMotherboard()

let amdCPU = amdFactory.createCPU()
let amdMotherboard = amdFactory.createMotherboard()

print(intelCPU.name) // 输出: Intel CPU
print(intelMotherboard.name) // 输出: Intel Motherboard
print(amdCPU.name) // 输出: AMD CPU
print(amdMotherboard.name) // 输出: AMD Motherboard

3. 工厂方法模式（Factory Method Pattern）

工厂方法模式（Factory Method Pattern）是一种常用的创建型设计模式，它定义了一个用于创建对象的接口，但是让子类决定将哪一个类实例化，工厂方法模式让一个类的实例化延迟到其子类。

工厂方法模式可以很好地解决创建对象的问题，可以避免直接使用 new 这种方式创建对象，而是通过工厂方法来创建对象，这样可以避免代码中的耦合问题，增强代码的可扩展性和可维护性。

下面是一个使用 Swift 5 实现工厂方法模式的示例代码：

  • Swift Code

    // 抽象产品
protocol Product {
    var name: String { get }
}

// 具体产品 A
class ProductA: Product {
    var name: String { return "Product A" }
}

// 具体产品 B
class ProductB: Product {
    var name: String { return "Product B" }
}

// 抽象工厂
protocol Factory {
    func createProduct() -> Product
}

// 具体工厂 A
class FactoryA: Factory {
    func createProduct() -> Product {
        return ProductA()
    }
}

// 具体工厂 B
class FactoryB: Factory {
    func createProduct() -> Product {
        return ProductB()
    }
}

// 客户端代码
let factoryA = FactoryA()
let factoryB = FactoryB()

let productA = factoryA.createProduct()
let productB = factoryB.createProduct()

print(productA.name) // "Product A"
print(productB.name) // "Product B"

在这个示例代码中，我们定义了一个 Product 协议，其中包含一个 name 属性，用于表示产品的名称。然后，我们定义了两个具体产品 ProductA 和 **ProductB**，它们分别实现了 Product 协议。

接着，我们定义了一个 Factory 协议，其中包含一个 createProduct 方法，用于创建产品。然后，我们定义了两个具体工厂 FactoryA 和 **FactoryB**，它们分别实现了 Factory 协议，用于创建不同的产品。

最后，在客户端代码中，我们实例化了两个具体工厂 factoryA 和 **factoryB**，然后使用它们的 createProduct 方法来创建具体产品 productA 和 **productB**。通过这种方式，我们可以避免在客户端代码中直接使用 new 来创建对象，而是通过工厂方法来创建对象，从而提高了代码的可扩展性和可维护性。

4. 建造者模式（Builder Pattern）

建造者模式（Builder Pattern）是一种创建型设计模式，它允许你创建复杂的对象，同时将其构造过程与表示分离。使用建造者模式可以将一个复杂对象的构建过程和表示分离，使得同样的构建过程可以创建不同的表示。

建造者模式适用于以下情况：

  • 当需要创建的对象具有复杂的内部结构时，可以使用建造者模式将构造方法的调用顺序与属性的设置分离，从而降低代码的复杂度。
  • 当需要构建不同属性的对象时，可以使用建造者模式来创建不同的属性组合。

以下是使用 Swift 5 实现建造者模式的示例代码：

  • Swift Code

    // Product
class Pizza {
    var dough: String = ""
    var sauce: String = ""
    var toppings: [String] = []

    func showPizza() {
        print("Pizza with \(dough) dough, \(sauce) sauce and toppings: \(toppings.joined(separator: ", "))")
    }
}

// Builder
protocol PizzaBuilder {
    func addDough()
    func addSauce()
    func addToppings()
    func getPizza() -> Pizza
}

// Concrete Builder
class MargheritaPizzaBuilder: PizzaBuilder {
    private var pizza = Pizza()

    func addDough() {
        pizza.dough = "Thin crust"
    }

    func addSauce() {
        pizza.sauce = "Tomato sauce"
    }

    func addToppings() {
        pizza.toppings = ["Mozzarella cheese", "Basil leaves"]
    }

    func getPizza() -> Pizza {
        return pizza
    }
}

// Director
class PizzaDirector {
    private var builder: PizzaBuilder

    init(builder: PizzaBuilder) {
        self.builder = builder
    }

    func makePizza() -> Pizza {
        builder.addDough()
        builder.addSauce()
        builder.addToppings()
        return builder.getPizza()
    }
}

// Client code
let margheritaBuilder = MargheritaPizzaBuilder()
let pizzaDirector = PizzaDirector(builder: margheritaBuilder)
let margheritaPizza = pizzaDirector.makePizza()
margheritaPizza.showPizza()

在这个示例中，**Pizza** 是我们要创建的复杂对象，而 PizzaBuilder 协议定义了建造者应该具备的方法，包括添加面团、酱料和配料等。 MargheritaPizzaBuilder 是建造者的具体实现，它实现了 PizzaBuilder 协议中的方法，用于构建玛格丽特比萨。**PizzaDirector** 类负责管理建造者的调用顺序，它根据特定的建造者构建产品。最后，客户端代码使用建造者和指挥者来构建产品并将其展示出来。

5. 原型模式（Prototype Pattern）

原型模式（Prototype Pattern）是一种创建型设计模式，它允许通过复制现有对象来创建新对象，而不必通过标准构造函数创建。该模式需要实现一个克隆方法，该方法克隆原型对象并返回副本。这使得客户端能够生成新对象，同时保留原始对象的状态。

原型模式通常用于创建复杂的对象，这些对象的创建过程比简单的构造函数调用要复杂得多。通常，创建这些对象需要许多步骤和复杂的算法，这些算法需要大量的计算和资源。在这种情况下，使用原型模式可以更有效地生成新对象，因为不需要在每次创建对象时重新计算它们。

下面是一个使用原型模式的简单示例，其中我们将克隆现有汽车对象以创建新对象。首先，我们需要创建一个“Car”协议，并实现克隆方法：

protocol Car: AnyObject, NSCopying {
    var model: String { get set }
    var color: String { get set }
    var price: Double { get set }
}

extension Car {
    func copy(with zone: NSZone? = nil) -> Any {
        return CarPrototype(model: model, color: color, price: price)
    }
}

class CarPrototype: Car {
    var model: String
    var color: String
    var price: Double

    init(model: String, color: String, price: Double) {
        self.model = model
        self.color = color
        self.price = price
    }
}

在上面的代码中，我们定义了“Car”协议，并使用“CarPrototype”类实现了该协议。我们还使用“NSCopying”协议为该类提供了克隆方法。

接下来，我们将创建一个“CarFactory”类，该类将提供一些现有汽车对象，供客户端使用克隆方法生成新对象：

class CarFactory {
    private var prototypes = [String: Car]()

    init() {
        prototypes["SUV"] = CarPrototype(model: "SUV", color: "Red", price: 50000)
        prototypes["Coupe"] = CarPrototype(model: "Coupe", color: "Blue", price: 80000)
    }

    func createCar(ofType type: String) -> Car? {
        return prototypes[type]?.copy() as? Car
    }
}

在上面的代码中，我们使用“prototypes”字典来存储现有的汽车对象。客户端可以使用“createCar(ofType:)”方法来获取现有汽车对象的副本，并生成新对象。

下面是一个示例，演示如何使用“CarFactory”类生成新的汽车对象：

let factory = CarFactory()
let suv1 = factory.createCar(ofType: "SUV")
let suv2 = factory.createCar(ofType: "SUV")

suv1?.model // SUV
suv1?.color // Red
suv1?.price // 50000

suv2?.model // SUV
suv2?.color // Red
suv2?.price // 50000

6. 单例模式（Singleton Pattern）

单例模式（Singleton Pattern）是一种创建型模式，它保证一个类只有一个实例，并提供一个全局访问点。

单例模式的常见应用场景包括：

  • 对于某些类，我们只需要有一个实例，比如系统的日志管理器、线程池、数据库连接池等；
  • 想要跨越多个模块访问某些数据，可以将数据存储在单例对象中，以便于跨模块访问。

在 Swift 中，可以使用静态变量或全局变量来实现单例模式，也可以使用结构体来实现。

以下是一个使用静态变量实现单例模式的例子：

class Singleton {
    static let shared = Singleton()

    private init() {
        // 私有化初始化方法，避免在外部创建实例
    }

    func someMethod() {
        // 单例类的其他方法
    }
}

在上面的例子中，我们使用一个名为 shared 的静态变量来保存唯一的实例。**shared** 变量在第一次访问时会触发 Singleton 类的初始化方法，而由于该方法是私有的，因此无法从外部创建实例。其他的方法则可以通过 Singleton.shared 来访问。由于 Swift 的静态变量是线程安全的，因此该实现方式可以保证单例模式的正确性。
